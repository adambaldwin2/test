buzhug can work on client-server mode, using an asynchronous HTTP server. If the database is used by several processes or threads, you should use this mode to ensure the consistency of the data

To use this mode :

  • start the script buzhug_server on the command line : 
    >python buzhug_server.py [-H host] [-P port]
    This will start an asynchronous HTTP server on the specified host and port. Default values are host = "localhost" and port = 3456

  • in the program using the database, instead of 
    from buzhug import Base
db = Base('teams')
    use the ProxyBase class : 
    from buzhug import ProxyBase
db = ProxyBase('teams'[,host=host[,port=port]])
You can use the ProxyBase instance exactly like a buzhug.Base instance. Method calls result in an HTTP request to the server ; this makes client-server mode slower than embedded mode
